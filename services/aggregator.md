# Synapse Metrics Aggregator

## Purpose

The Metrics Aggregator acts as the central hub of the Synapse system.

It collects data from all trackers, generates session statistics, and produces a single unified metrics packet.

The Aggregator does NOT calculate scores.

Its responsibility is only to gather and organize data.

---

## Architecture

KeyboardTracker
MouseTracker
WindowTracker
        ↓
MetricsAggregator
        ↓
Unified Metrics Packet
        ↓
Scoring Engine

---

## Data Sources

### Keyboard Tracker

Provides:

- Keys Per Minute
- Backspaces Per Minute
- Average Pause
- Long Pause Count
- Typing Variance

---

### Mouse Tracker

Provides:

- Mouse Speed
- Click Rate
- Movement Distance

---

### Window Tracker

Provides:

- Active Window
- Window Switch Count

---

### Session Metrics

Generated by the Aggregator.

Provides:

- Session Duration
- Break Count
- Time Since Last Break

---

## Break Detection

A break is detected when:

No Keyboard Activity
AND
No Mouse Activity

for more than 5 minutes.

When this occurs:

break_count += 1

The Aggregator prevents duplicate break counting during a single idle period.

## Why This Exists

The Scoring Engine should never communicate directly with trackers.

Instead:

Trackers
→ Aggregator
→ Scoring Engine

This creates a clean architecture and allows new trackers to be added later without changing scoring logic.

---

## Future Integrations

Planned additions:

- Blink Detection
- Posture Detection
- Gaze Tracking
- Emotion Signals

These modules will simply add new metrics to the Aggregator output.

No scoring code will need to change.
